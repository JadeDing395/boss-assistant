## 1. 安装：3 步搞定（macOS）

> 适用：macOS 10.15 (Catalina) 或更新。装的是「Chrome 扩展 + Mac 后台点击器」完整版。

### 第 1 步：下载 + 解压

1. 到项目主页下载 `boss-assistant-v1.1.1-macOS.zip`
2. 解压（双击 zip 文件即可）
3. 把解压后的文件夹放到一个**固定位置**（推荐：`~/Documents/` 或 `~/Applications/`，**别放下载文件夹里**，下载夹会被你定期清理）

### 第 2 步：双击 `install.command`

打开解压后的文件夹，找到 **`install.command`**，双击运行。

如果 macOS 弹「无法打开，未识别开发者」：
- 在 Finder 里**右键**该文件 → 选「打开」
- 弹窗里再点「打开」
- 这是 macOS 给非签名脚本的安全提示，只有第一次会问

终端会弹出来，全程中文交互。会自动完成：
- ✓ 装 Python 依赖（如果你没装 Python，会引导你装）
- ✓ 配置 Mac 后台点击器开机自启（macOS LaunchAgent）
- ✓ 打开 Chrome 扩展页 + macOS 辅助功能设置，提示你授权

跟着提示点几下「确认」就行，**预计 2-3 分钟**。

### 第 3 步：加载 Chrome 扩展

`install.command` 跑完后会自动打开 `chrome://extensions/`，按提示：
1. 右上角打开「**开发者模式**」
2. 点「**加载已解压的扩展程序**」
3. 选解压目录里的 **`boss-assistant/`** 文件夹（**不是整个 zip 解压根目录**）

加载成功后，Chrome 右上角会多出一个插件图标（拼图样式）。点一下，可以把它「固定」（pin）到地址栏右侧，方便随时打开。

✅ 完成。现在打开任意 Boss 直聘页面，点扩展图标，侧边栏会弹出来。

## 2. 安装：仅 Chrome 扩展（Windows / Linux）

> Windows / Linux 用户无法用 Mac 后台点击器（OS 级真鼠标），只能用浏览器内合成事件方式，**触发 Boss 风控的概率略高**。建议这类用户优先用「🛡 低风险模式」。

### 步骤

1. 下载 + 解压 `boss-assistant-v1.1.1-macOS.zip`（虽然名字带 macOS，扩展部分跨平台）
2. 打开 Chrome / Edge / Brave：`chrome://extensions/`
3. 右上角打开「**开发者模式**」
4. 点「**加载已解压的扩展程序**」
5. 选解压后里面的 **`boss-assistant/`** 子文件夹（不是 zip 根目录）
6. 加载后在 Boss 直聘网页点扩展图标即可

> 不用碰 `claude-boss-clicker/` 文件夹和 `install.command`，那是给 macOS 用的。

## 3. 配置：AI Key（必填）

插件靠 AI 评分判断候选人匹配度，所以必须配 OpenAI 兼容接口的 API Key。

### 推荐 API 提供商（任选一家）

| 提供商 | 优点 | 注册 |
|---|---|---|
| **DeepSeek**（推荐） | 国内访问稳定，便宜（百万 token ¥1-2） | https://platform.deepseek.com/ |
| **硅基流动 SiliconFlow** | 同上，模型选择多 | https://siliconflow.cn/ |
| **智谱 BigModel** | 国产合规，GLM 系列 | https://bigmodel.cn/ |
| **OpenAI** | 海外，效果最好 | https://platform.openai.com/ |

### 配置步骤

1. 打开侧边栏 → 展开「**筛选条件 → AI 配置（OpenAI 兼容）**」
2. **常用网关**：从下拉选你的提供商（会自动填好 BaseURL）
3. **常用模型**：选推荐模型（DeepSeek 推荐 `deepseek-chat`，其他类似）
4. **apiKey**：粘贴你刚拿到的 key（密码框，输入时不显示）
5. 右上角点 **「保存」**（橙色高亮就是有未保存的改动）

✅ 配好后下方会显示 `Key：已保存（不回显）` —— 安全起见，保存后不会把 key 显示回界面，但实际已存在本地。

> ⚠️ Key 只存在你**本机**的 chrome.storage.local，不会上传到任何服务器。

## 4. 配置：岗位 + 筛选条件

### 4.1 选岗位

侧边栏顶部「**岗位**」下拉 —— 从你 Boss 「**职位管理**」自动同步过来的岗位列表。

> 下拉是空的？先去 Boss 网页左侧菜单点开「**职位管理**」一次（让页面加载岗位数据），刷新插件即可看到。

### 4.2 通过阈值

旁边的「**通过阈值**」输入框 —— AI 评分 ≥ 这个分数才算通过。
- 0-100 之间
- 建议 **70-80** 起步
- 太低会通过太多人，太高可能漏掉合适的；先用 75 跑一批观察

### 4.3 关键词 + 硬过滤

展开「**筛选条件 → 寻访模式 · 关键词 · 年龄学历**」：

**寻访模式**：
- **AI 模式**（推荐）：用大模型对"简历 vs JD"语义评分
- **无 AI 模式**：纯关键词匹配，不调 AI（省钱但准确度差）

**关键词组合公式**：
```
通过 = (必含全 AND 命中) AND (任意 OR 命中其一) AND NOT 排除
```

- **必含关键词**（AND）：必须全部出现在简历里。例：`插画`、`二次元`
- **任意关键词**（OR）：命中其中任何一个就算通过。例：`角色KV/宣传图/海报绘制`
- **加分项**（AI 模式专属）：不影响通过/不通过，命中加分。例：`八大美院/大厂经验/上线项目`
- **排除关键词**：命中即淘汰。例：`外包/销售/外派`

> AI 会语义扩展关键词，比如写「欧卡」会自动匹配「欧美卡通/迪士尼/写实」，写「大厂」会匹配腾讯/网易/米哈游/字节/B 站/完美世界等。

**硬过滤（最优先级，命中直接跳过，不调 AI）**：
- **年龄区间**：例如填 25-35，<25 或 >35 直接跳。0 = 不启用
- **学历 / 院校**：本科及以上 / 硕士 / 985 / 211 / 八大美院等
- **最近 Gap 上限（月）**：候选人最新一段工作的结束日期距今超 N 个月直接淘汰。写"至今"视为在职。0 = 不启用

> 这些是按岗位独立保存的，切到别的岗位会用别的设置。

### 4.4 岗位要求（JD）

展开「**筛选条件 → 岗位要求（JD）**」，粘贴 JD 全文。
- AI 模式下，JD + 候选人简历会一起送给 AI 评分
- 如果 JD 写得清楚（明确技能、必须项、加分项），AI 判得更准

## 5. 选择运行模式：低风险 / 半自动 / 全自动

展开「**进阶运行设置 → 本机点击器**」，里面有「**运行模式**」三档选择器。

| 模式 | 行为 | 风险 | 适合 |
|---|---|---|---|
| 🛡 **低风险** | 自动筛人 + 生成草稿，**不自动点任何按钮**。你看到「待人工」列表后**手动**点打招呼 | 极低 | **默认 + 推荐**。新号、工作号、刚开始用 |
| ⚡ **半自动** | 自动点「打招呼」+ 发模板，但**每次点击前在本机弹确认**（你随时可 ESC 取消） | 中等 | 已经校准好筛选规则，想省点手点动作 |
| ⚠️ **全自动** | 完全无人值守，自动点 + 不弹确认 | **高** | 老熟号 + 严格小批量（≤30/次）|

切换会有二次确认弹窗。**强烈建议从「低风险」开始用 1-2 周观察 AI 评分准确度，再考虑切到半自动**。

### 5.1 「主动寻访」与「自动回复」在三种模式下的具体行为（v1.1.2 重新整理）

| 流程 | 🛡 低风险 | ⚡ 半自动 | ⚠️ 全自动 |
|---|---|---|---|
| **主动寻访**（推荐牛人页逐个评估 + 招呼） | 只评分 + 写日志，**不点任何按钮** | 自动点「打招呼」，每次点前本机弹确认（ESC 可取消） | 自动点「打招呼」，无确认 |
| **自动回复**（沟通页未读会话） | AI 评估 → **dispatchEvent 点「继续沟通」**（仅 UI 视图切换，不属业务操作）→ 输入框填好回复草稿（**不发送**） | AI 评估 → 真鼠标点「继续沟通」→ 输入框填好回复草稿（**不发送**，留人工核对） | AI 评估 → 真鼠标点「继续沟通」→ 填好回复 → **自动点发送** → 切下一位 |

**关键变化（v1.1.2）**：自动回复不论哪种模式都会主动点一下「继续沟通」按钮（boss 简历面板里那颗），因为不点这颗按钮根本不会出现输入框，草稿都填不进去。这颗按钮只是 UI 视图切换（简历 → 聊天），不是高危业务操作，所以低风险下也会执行。

### 5.2 自动回复的待人工汇总（v1.1.2 新增点击跳转）

在「自动回复」页底部「**待人工处理汇总**」里，点任意一张候选人卡片：
- 自动复制候选人姓名到剪贴板
- 自动切到 Boss 沟通页
- 你 ⌘V 粘贴搜索即可定位到该候选人会话

## 6. 开始筛选 + 查看进度

### 6.1 开始

1. 打开 Boss 「**推荐牛人**」页面（或精选 / 最新子页）
2. 侧边栏底部点 **「开始」**（蓝色按钮）
3. 插件会自动切到你选的岗位 → 一个一个看候选人 → AI 判断 → 标记「待人工」/ 「不匹配」

### 6.2 实时观察

侧边栏顶部「**状态栏**」：
- 已处理 / 通过 / 待人工 实时计数
- 「**风险**」徽章显示当前模式
- 「**模式**」显示 AI/关键词 + 寻访/回复

底部「**运行日志**」实时输出每个候选人的判断结果、AI 评分、跳过原因等。

### 6.3 停止

点 **「停止」**（顶部状态栏右上角的红色 ⏹ 也能停）。

> 已经被 AI 评过的候选人会自动跳过，不重复消耗 token。每轮跑完日志会写「**本次共跳过 X 个上一轮已评过的候选人，节省了 X 次 AI 调用**」。

## 7. 查看历史 + 导出 CSV

切到「**历史/统计**」标签：

### 7.1 统计面板

6 个数字卡片：总记录 / 今日 / 通过率 / 平均分 / 待人工 / 硬过滤淘汰。**数字会根据下方"日期范围"实时变化**。

### 7.2 筛选 + 搜索

- **状态**：全部 / 仅通过 / 仅未通过 / 仅待人工
- **日期范围**：今日 / 近 7 天 / 近 30 天（默认）/ 全部
- **关键词**：姓名 / 原因模糊搜

### 7.3 历史卡片

每条记录显示：
- 姓名 + 结论（**推荐 / 不匹配 / 待人工**）+ 分数 + 时间
- 推荐 / 淘汰理由（分点列出）
- 当时所属岗位
- ID（点击复制）
- **「在 Boss 中定位 →」** ← 点这里可以跳回 Boss 网页 + 自动滚到该候选人卡片 + 金色高亮 4 秒

> 如果该候选人已经被 Boss 推荐列表刷出去，定位会失败，卡片会自动置灰 + 显示「已不在 Boss 推荐流」。重开侧边栏可重试。

### 7.4 导出 CSV

右上角「**导出 CSV**」按钮 —— 按当前的「日期范围 + 状态 + 关键词」筛选条件导出。
- 文件名带日期范围标识：`claude-boss-history-30d-20260527.csv`
- 字段：时间 / 岗位 / 姓名 / ID / 评分 / 原因 / 决策 / 待人工 / 草稿
- 直接 Excel 打开（UTF-8 编码）

### 7.5 数据保留期

本地最多保留近 **30 天** 的评分记录，更早的在下次运行时自动清理。**所有数据只在本机，不上传服务器。**

## 8. 本机点击器：什么时候用，怎么用

### 8.1 干啥的

仅 macOS 可用的 Python 后台服务，提供 **OS 级真鼠标点击** —— 绕开 Boss 的 `isTrusted` 检测。在「半自动 / 全自动」模式下会被调用。

### 8.2 检查是否在跑

侧边栏「**进阶运行设置 → 本机点击器**」面板右上角：
- 🟢 **已连通 v1.0** = 在跑 ✓
- 🔴 **未连通** = 没跑

### 8.3 没在跑怎么办

如果你装的时候双击过 `install.command`，那 clicker 已经配置成开机自启 —— 一般不会出现"没跑"的情况。

**真出现了**（被你手动 kill / 系统报错），切到半自动/全自动时会自动弹一个对话框：

- 【确定】= 复制重启命令到剪贴板 →  
  打开终端 ⌘V + 回车，命令是：`launchctl kickstart -k gui/$UID/com.claude-boss.clicker`
- 【取消】= 提示去重新双击 `install.command`（如果你从来没装过开机自启）

### 8.4 一次性永久配置

如果你跳过了 `install.command` 安装步骤，单独跑一次：

1. Finder 进入 `claude-boss-clicker/` 文件夹
2. 双击 **`install_autostart.command`**
3. 跟着提示授权（会弹辅助功能权限请求）
4. 完成后 clicker 永久在后台跑，开机自启

### 8.5 卸载点击器

双击 `claude-boss-clicker/uninstall_autostart.command` —— 移除开机自启服务，清理临时文件。

## 9. 常见问题 FAQ

### Q1：装好扩展，打开 Boss 没反应？

- 确认你在 `*.zhipin.com` 域名下
- 刷新 Boss 页面（⌘R）一次让 content script 重新注入
- 检查扩展是否被 disable：`chrome://extensions/` 看开关是不是开的

### Q2：AI 评分一直不通过 / 全是 0 分？

- 检查 AI 配置是否填全（BaseURL + apiKey + model）
- 看运行日志有没有「AI 失败」或「AI 返回异常」字样 —— 通常是 Key 无效或余额不足
- 用「测试连通」按钮（暂无；先看日志诊断）

### Q3：「岗位」下拉是空的？

去 Boss 左侧菜单点「**职位管理**」一次让页面加载岗位列表，然后回到插件刷新（点开始或关闭重开侧边栏）。

### Q4：跑了一次后第二次跑，AI 又把同样的人评了一遍？

- 1.1.1 已修复此问题，会按候选人身份 + 岗位自动跳过
- 跑完会写「跳过 X 人，节省 X 次 AI 调用」到日志
- 还重复？告诉维护者，附日志截图

### Q5：本机点击器一直「未连通」？

- 看「[本机点击器：没在跑怎么办](#83-没在跑怎么办)」
- 或终端跑：`launchctl list | grep claude-boss` —— 没输出说明 LaunchAgent 没装，去跑 `install_autostart.command`

### Q6：候选人卡片点「在 Boss 中定位 →」没反应？

- 候选人可能已被 Boss 推荐流刷出去（动态列表）
- 插件会自动尝试 3 个子页（推荐 / 精选 / 最新）+ 滚动加载
- 都找不到 → 卡片自动置灰 +「已不在 Boss 推荐流」。重开侧边栏可重试

### Q7：「Gap 上限」怎么算？

候选人简历里**最新一段工作**的结束日期 vs 今天。
- 写「至今 / present / now」= 在职，不淘汰
- 写「2024.05」（具体年月）= 算月数差

### Q8：触发了 Boss「操作过于频繁」？

**立即停止，今天别再跑了**。建议：
- 「运行参数」里把「随机延迟」拉大（最大值改成 5000+ ms）
- 「每次最多联系人数」压低（≤10）
- 切回「🛡 低风险」模式跑几天
- 「仿人节奏」开到「强」

### Q9：想清空历史重新开始？

「历史/统计」页右上角「**清空**」按钮。只清插件本地评分记录，不影响 Boss 任何数据。

### Q10：换 Chrome / 换电脑了，怎么把配置带过去？

目前需要重新配。后续可能加导入/导出配置功能。

## 10. 卸载

1. **移除 Chrome 扩展**：`chrome://extensions/` → 找到「招聘 Assistant-Boss」→ 移除
2. **卸载 Mac 后台点击器**：双击 `claude-boss-clicker/uninstall_autostart.command`
3. **清理 LaunchAgent**（如果上一步没成功）：
   ```bash
   launchctl unload ~/Library/LaunchAgents/com.claude-boss.clicker.plist
   rm ~/Library/LaunchAgents/com.claude-boss.clicker.plist
   ```
4. 删掉整个解压文件夹

完事，无残留。

## 11. ⚠️ 风险与注意事项

**此工具违反 Boss 直聘服务条款，账号有被风控 / 封禁的风险。请自行承担风险。**

### 强烈建议

1. **用工作账号慎用，新号尤其慎用**
2. **第一次跑量级压到 ≤5**，观察是否触发风控
3. **任何"操作过于频繁"提示出现立即停**
4. **不要 24 小时不间断运行**
5. **不要在公开渠道传播你的使用截图 / 经验**
6. **优先用「🛡 低风险」模式** —— 已经省 80% 工作量，安全很多
7. **切到 auto 模式前**：先用 1-2 周低风险跑一批，确认 AI 判断准确度

### 数据隐私

- 所有评分历史、API Key、岗位配置都仅存你**本机** chrome.storage.local
- 插件**不会上传任何数据到任何服务器**
- 你的 AI API 调用是直接从你浏览器到 AI 提供商，中间没经过我们
